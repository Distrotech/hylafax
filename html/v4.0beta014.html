HYLAFAX CHANGE INFORMATION
    Current Version: v4.0beta014
    Previous Version: v4.0beta013
    Master FTP Site: sgi.com (192.48.153.1), directory sgi/fax

Note that this is a major new distribution and lots of things have changed. Consult the general overview for a high-level description of the new functionality provided in the v4.0 software.

This document describes the changes made to the software between the previous and current versions (see above). If you don't find something listed here, then it was not done in this timeframe, or it was not considered important enough to be mentioned. The following information is located here:

  • Changes in the software configuration
  • Changes to faxq
  • Changes to faxsend
  • Changes to pagesend
  • Changes to faxgetty
  • Changes to hfaxd
  • Changes to faxalter
  • Changes to sendpage
  • Changes to faxsetup
  • Changes to faxmail
  • Changes in miscellaneous stuff

CHANGES IN THE SOFTWARE CONFIGURATION:
  • configure now verifies the version of the configured TIFF software

CHANGES TO FAXQ:
  • the filenames for continuation cover pages were made to follow the document naming conventions so that they are properly handled by the document reference counting logic

CHANGES TO FAXSEND:
  • time spent connected to a remote device is now recorded in addition to the total time spent processing a job
  • a bug was corrected in the processing of jobs that get a communication error that causes an incomplete document to be sent
  • the Class1RecvIdentTimer configuration parameter is now correctly parsed

CHANGES TO PAGESEND:
  • time spent connected to a remote device is now recorded in addition to the total time spent processing a job

CHANGES TO FAXGETTY:
  • a FreeBSD compilation problem was fixed
  • a new configuration parameter ModemReadyState was added to statically define the modem state to pass to faxq when a modem becomes ready for use

CHANGES TO HFAXD:
  • support was added to map between fax UIDs and symbolic names
  • setting job parameters that have side effects is now handled properly
  • the SNPP support was changed to workaround a Solaris 2.5 bug that caused binding to a privileged port to fail
  • a bug was fixed in the event logging support that caused hfaxd to dump core

CHANGES TO FAXALTER:
  • the handling of time values was corrected

CHANGES TO SENDPAGE:
  • the default SNPP server is now intialized to be on the ``localhost''

CHANGES TO FAXSETUP:
  • fixups for Solaris systems

CHANGES IN MISCELLANEOUS STUFF:
  • the xferstats, recvstats, and faxcron scripts were updated to handle the new xferlog format
  • the dial rules files now map alphabetic characters to their numeric equivalents when forming a string to pass to the modem
  • the calling sequence to parseAtSyntax was changed to avoid problems with callers not allocating enough space to return an error message
