HYLAFAX CHANGE INFORMATION
    Current Version: v4.0beta006
    Previous Version: v4.0beta005
    Master FTP Site: sgi.com (192.48.153.1), directory sgi/fax

Note that this is a major new distribution and lots of things have changed. Consult the general overview for a high-level description of the new functionality provided in the v4.0 software.

This document describes the changes made to the software between the previous and current versions (see above). If you don't find something listed here, then it was not done in this timeframe, or it was not considered important enough to be mentioned. The following information is located here:

  • General changes in the software
  • Changes in the software configuration
  • Changes in the modem config files
  • Changes to hfaxd
  • Changes to pagesend
  • Changes to sendpage
  • Changes to faxmail
  • Changes in the support library

GENERAL CHANGES IN THE SOFTWARE:
  • the default port for the client-server protocol has moved from 4559 to 4558

CHANGES IN THE SOFTWARE CONFIGURATION:
  • relative pathnames specified for the TIFFINC and LIBTIFF configuration parameters are now adjusted to reflect their use within subdirectories
  • the logic to extract the gcc version string no longer is confused by local prefixes (e.g. cygnus-2.7.2)
  • configure now checks to see if the random function should be emulated
  • the logic to check if memory-mapped files are supported was corrected
  • the default function prototypes for various socket calls have been changed to use struct sockaddr* parameters
  • the default function prototype for crypt was corrected
  • configure now checks if the crypt function is to be found in a -lcrypt library

CHANGES IN THE MODEM CONFIG FILES:
  • the ZyXEL 1496 configuration files disable ECM to avoid firmware problems

CHANGES TO HFAXD:
  • a bug that caused hfaxd to detach itself from the controlling terminal when started with the -I or -O options was fixed
  • the LASTTIME job parameter is now set correctly

CHANGES TO PAGESEND:
  • a new PagerTTYParity parameter was added to the info database to specify if the IXO/TAP service provider requires a non-standard bits+parity setup

CHANGES TO SENDPAGE:
  • sendpage is still broken

CHANGES TO FAXMAIL:
  • faxmail was rewritten to use the new TextFmt class for converting ASCII text to PostScript
  • the default formatting was changed
  • new support was added to control most formatting through header fields in the envelope of the parsed mail

CHANGES IN THE SUPPORT LIBRARY:
  • zlib was updated from version 0.95 to 0.99 (nominally called 1.0)
  • portability issues with sockets have been isolated in a new class
  • a bug in the transmission of the SENDTIME job parameter was fixed
  • a new TextFmt class was added to encapsulate the conversion of ASCII text to PostScript
  • a bug in the parsing of fax server specification strings was corrected
  • various portability problems were fixed
  • faxinfo now reports the correct page dimensions for pages that are not found in the page size database
  • faxrcvd no longer prints duplicate information
  • textfmt was rewritten to use the TextFmt class
  • xferfaxstats now prints the correct column heading for the sort key
