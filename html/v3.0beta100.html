HYLAFAX CHANGE INFORMATION
    Current Version: v3.0beta100
    Previous Version: v3.0beta096
    Master FTP Site: sgi.com (192.48.153.1), directory sgi/fax

This document describes the changes made to the software between the previous and current versions (see above). If you don't find something listed here, then it was not done in this timeframe, or it was not considered important enough to be mentioned. The following information is located here:

  • General changes
  • Changes in the software configuration
  • Changes in the modem config files
  • Changes to pagesend
  • Changes to faxaddmodem
  • Changes in the utility code

GENERAL CHANGES:
  • the IRIX binary distribution inst images were overhauled; HylaFAX packages now replace old FlexFAx packages, the PostScript RIP is now broken out into a separate package (to make it easy to select between DPS-based and GS-based RIPs), the HTML materials were renamed, and the sample AFM files were placed in the software image
  • the HTML materials were significantly revised (but are still unfinished)

CHANGES IN THE SOFTWARE CONFIGURATION:
  • various problems with configuring optional packages were corrected
  • the HTML materials are now processed by configure to fix pathnames
  • the DPS and Ghostscript-based RIPS are now handled separately so that IRIX systems can configure each independently
  • parameters were added for controlling the use of -lmalloc (LIBMALLOC) and -lsun (LIBSUN)
  • parameters were added for configuring the HTML materials; consult the config.site file
  • configure was changed to deal with the fact that various sed's only support 100 commands
  • strdup is now checked for in -lg++ to deal with some Sun systems
  • a problem with deducing signal handler types was corrected
  • the default GETTY definition for Ultrix was corrected
  • the manual page processing was corrected for various systems
  • the install.sh script used to emulate the IRIX install program now understands that certain files should be updated only if they have not been changed since the last distribution; also a problem with stripping non-executable files has been corrected

CHANGES IN THE MODEM CONFIG FILES:
  • typos were corrected in the PM14400FXMT and ZyXEL Class 2.0 files

CHANGES TO PAGESEND:
  • the tty device is now setup to use 7-bit, even parity, per the spec

CHANGES TO FAXADDMODEM:
  • existing inetd.conf entries are now checked to make sure the pathname for faxd.recv corresponds to the path configured for installation
  • old prototype config files are ignored
  • the DTE-DCE line speed is checked against the speed specified by the ModemRate parameter and if they are different the user is prompted to verify

CHANGES IN THE UTILITY CODE:
  • the faxrcvd script now can dispatch received facsimile from well-known senders automatically; the facsimile is sent by electronic mail as a MIME-encoded PostScript document
  • the ps2fax shell scripts no longer try to workaround certain invalid Windows-generated PostScript constructs (the patch had busted valid PostScript)
