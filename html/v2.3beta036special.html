FLEXFAX CHANGE INFORMATION
    Current Version: v2.3beta036special
    Previous Version: v2.3beta036
    Master FTP Site: sgi.com (192.48.153.1), directory sgi/fax

This document describes the changes made to the software between the previous and current versions (see above). If you don't find something listed here, then it was not done in this timeframe, or it was not considered important enough to be mentioned (but just about everything is mentioned).

This was a special distribution made to correct a problem in the Class 2.0 driver. There were a few other minor problems corrected at the same time, but otherwise the software is the same as v2.3beta036; consult the change information for v2.3beta036 for details.

This distribution contains the following changes:

  • Linux-related changes:
    • change to the configure script
    • change to etc/faxaddmodem.sh and etc/probemodem.sh
    • changes in port/linux for filesystem reorganization
  • change to etc/faxaddmodem.sh and etc/probemodem.sh so that Solaris 2.x is treated as SVR4 (in order to get appropriate UUCP lock files)
  • fix to the Class 2.0 driver to send RTC at the end of each page transmitted
  • fix to faxd/FaxRequest.c++ to correct notification of transmit jobs
  • fix to the fax server's Group 3 decoder so that copy quality checking of 2D-encoded rows that exceed the line length are not treated as errors (the G3 decoder appears to incorrectly count pixels in this case)
  • fix to recvfax/auth.c so that comment-only lines are not interpreted as ``accept anyone''
  • fix to util/FaxClient.c++ so that default port numbers are properly byte-swapped on Intel CPUs
  • change to util/SendFaxClient.c++ to remove the permission check on the -f option (a la sendmail 8.x)
  • fix to util/SendFaxClient.c++ so that UUCP-style mail addresses are not concatenated with the hostname (e.g. a!b!c!user@host)
