HYLAFAX CHANGE INFORMATION
    Current Version: v4.0beta010
    Previous Version: v4.0beta009
    Master FTP Site: sgi.com (192.48.153.1), directory sgi/fax

Note that this is a major new distribution and lots of things have changed. Consult the general overview for a high-level description of the new functionality provided in the v4.0 software.

This document describes the changes made to the software between the previous and current versions (see above). If you don't find something listed here, then it was not done in this timeframe, or it was not considered important enough to be mentioned. The following information is located here:

  • General changes
  • Changes in the software configuration
  • Changes to hfaxd
  • Changes to sendpage
  • Changes in miscellaneous stuff

GENERAL CHANGES:
  • support for sending pages has returned; there is a new version of sendpage and significant new additions to hfaxd (see below)

CHANGES IN THE SOFTWARE CONFIGURATION:
  • configure now emits the right definition for CONFIG_BADEXECVEPROTO for AIX systems
  • the check for support for memory-mapped files was corrected

CHANGES TO HFAXD:
  • support was added for the Simple Network Pager Protocol (SNPP)
  • support was added for filesystems that use the BSD-style semantics for setting the group ID on created files; hfaxd dynamically determines if the underlying filesystem for the spooling area has BSD-style or System V-style semantics and acts accordingly
  • a bug was fixed whereby changes to files were not reflected back to the information stored in the file cache
  • a bug that caused the STOR command to fail when the destination file did not exist has been fixed
  • the NLST command has been implemented
  • a new flag was added to control tracing of configuration file processing
  • a new job format control, %X, was added for returning the type of a job as a one character symbol
  • a race condition in the JWAIT command was fixed
  • a workaround was added for compiling with the HP-UX C++ compiler
  • hfaxd no longer aborts when /dev/tcp is found but /etc/netconfig is not found (required for correct operation on some versions of SunOS)

CHANGES TO SENDPAGE:
  • sendpage was rewritten to support the Simple Network Pager Protocol (SNPP) as defined in RFC 1861 (with some additions)
  • the command-line options have changed; consult the manual page

CHANGES IN MISCELLANEOUS STUFF:
  • the dialstring rules files now map alphabetics to their equivalent numeric key codes
  • the System V bootup script etc/hylafax has been changed to enable SNPP support by default
  • tracing functionality was added to the configuration file support used by all client and server applications
  • a bug was fixed in the notify.awk script that could cause it to go into an infinite loop if certain files were missing
  • a syntax error in the notify.awk script was corrected
