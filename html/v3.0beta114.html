HYLAFAX CHANGE INFORMATION
    Current Version: v3.0pl1 (aka v3.0beta114)
    Previous Version: v3.0pl0
    Master FTP Site: sgi.com (192.48.153.1), directory sgi/fax

This document describes the changes made to the software between the previous and current versions (see above). If you don't find something listed here, then it was not done in this timeframe, or it was not considered important enough to be mentioned.

This was a patch distribution made to correct problems found after the v3.0pl0 release. The main reason for this patch was to change certain C++ language constructs to follow recent changes to the ANSI C++ Reference Manual (ARM); this was important because GNU gcc release 2.7.0, by default, enforced these changes causing confusion among users.

The following information is located here:

  • Changes in the software configuration
  • Changes to faxq
  • Changes to faxsend
  • Changes to faxd.recv
  • Changes to textfmt

CHANGES IN THE SOFTWARE CONFIGURATION:
  • support was added for IRIX 6.x releases
  • -liberty is now used when the standard C library does not include the memmove or strdup functions
  • varous fixes for supporting GNU gcc version 2.7.0

CHANGES TO FAXQ:
  • a bug was fixed that caused job priorities to have no effect
  • a workaround was added for problems with send-only configurations with modems that did not support 2D-encoded data (setup restricted capabilities for modems specified on the fax command line with the -m option)

CHANGES TO FAXSEND:
  • change to ignore blank lines in configuration files
  • change to include line numbers in diagnostics generated while processing configuration files
  • portability fix for systems where mode_t is 16-bits

CHANGES TO FAXD.RECV:
  • a bug was fixed whereby shutdown files were being ignored when associated faxgetty processes were not running
  • a bug was fixed in the handling of job sequence numbers

CHANGES TO TEXTFMT:
  • a bug was fixed whereby N-up-style formatting did not work correctly
