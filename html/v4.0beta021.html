HYLAFAX CHANGE INFORMATION
    Current Version: v4.0beta021 (aka v4.0pl0)
    Previous Version: v4.0beta020
    Master FTP Site: sgi.com (192.48.153.1), directory sgi/fax

Note that this is a major new distribution and lots of things have changed. Consult the general overview for a high-level description of the new functionality provided in the v4.0 software.

This document describes the changes made to the software between the previous and current versions (see above). If you don't find something listed here, then it was not done in this timeframe, or it was not considered important enough to be mentioned. The following information is located here:

  • Changes in the software configuration
  • Changes to faxgetty
  • Changes to faxsetup
  • Changes in miscellaneous stuff

CHANGES IN THE SOFTWARE CONFIGURATION:
  • configuration requirements for SCO 5.0 are now handled automatically
  • configure now handles systems that use #defines for the syslog and vsyslog functions (e.g. SCO OS 5)
  • the pathname to the so_locations file used in creating DSOs under IRIX was corrected; this fixes build problems with the MipsPRO 7.x compilers under IRIX 6.2

CHANGES TO FAXGETTY:
  • a bug in the decoding of 2D-encoded facsimile page data was fixed

CHANGES TO FAXSETUP:
  • the -spool and -tiffbin options were removed in favor of a more general ``-with-VAR=[value]'' syntax; consult the manual page for details
  • faxsetup now forces the setting of the link to the ps2fax shell script used to image outbound PostScript documents
  • the construct that caused IRIX's /bin/sh to go into a infinite loop on exit was removed

CHANGES IN MISCELLANEOUS STUFF:
  • a workaround for a problem in the HP-UX ANSI C compilation environment was added to the string support code
  • the handling of escape codes in modem command strings was redone; this fixes several bugs, most notably the handling of strings with embedded line breaks followed by a <waitfor> request
  • the wedged script now checks if /etc/inittab is present before trying to disable runaway faxgetty processes
