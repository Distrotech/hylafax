HYLAFAX CHANGE INFORMATION
    Current Version: v4.0beta008
    Previous Version: v4.0beta007
    Master FTP Site: sgi.com (192.48.153.1), directory sgi/fax

Note that this is a major new distribution and lots of things have changed. Consult the general overview for a high-level description of the new functionality provided in the v4.0 software.

This document describes the changes made to the software between the previous and current versions (see above). If you don't find something listed here, then it was not done in this timeframe, or it was not considered important enough to be mentioned. The following information is located here:

  • Changes in the software configuration
  • Changes to hfaxd
  • Changes to faxmail
  • Changes to sendpage
  • Changes in the support library

CHANGES IN THE SOFTWARE CONFIGURATION:
  • the pathname for the vgetty program can now be configured with the PATH_VGETTY configuration parameter
  • the pathname for the directory where the TIFF tools are installed can now be configured with the TIFFBIN configuration parameter
  • the test program used to check for memory-mapped file support now uses a test that reflects actual usage (i.e. it verifies only that read-only support is present)
  • new checks for dev_t type and mknod system call
  • new checks for locating installed Ghostscript on Linux systems

CHANGES TO HFAXD:
  • hfaxd now automatically creates /dev/null, /dev/tcp, and /etc/netconfig files in the chroot'd filesystem on systems that appear to need them for proper operation
  • hfaxd now closes only the standard input, output, and error descriptors when detaching itself from the controlling terminal
  • portability problems associated with reusing a varargs parameter list twice without an intervening va_end/va_start have been corrected
  • missing includes of <net/errno.h> have been fixed

CHANGES TO FAXMAIL:
  • meta-headers were added to control which header lines are kept from the envelope of processed mail messages
  • the sample PostScript handwriting fonts have been removed
  • old comments about the -f option to sendfax that are no longer true were removed from the mailfax shell scripts
  • the manual page has been updated to reflect current usage

CHANGES TO SENDPAGE:
  • sendpage is still broken

CHANGES IN THE SUPPORT LIBRARY:
  • all uses to the gethostbyname function are now type-coerced with a new entry in the Socket class
