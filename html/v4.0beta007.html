HYLAFAX CHANGE INFORMATION
    Current Version: v4.0beta007
    Previous Version: v4.0beta006
    Master FTP Site: sgi.com (192.48.153.1), directory sgi/fax

Note that this is a major new distribution and lots of things have changed. Consult the general overview for a high-level description of the new functionality provided in the v4.0 software.

This document describes the changes made to the software between the previous and current versions (see above). If you don't find something listed here, then it was not done in this timeframe, or it was not considered important enough to be mentioned. The following information is located here:

  • Changes in the software configuration
  • Changes in the modem config files
  • Changes to faxsend
  • Changes to hfaxd
  • Changes to sendpage
  • Changes to miscellaneous programs
  • Changes in the support library

CHANGES IN THE SOFTWARE CONFIGURATION:
  • config.guess now recognizes ISC systems
  • a typo in config.h.in was corrected and by default support for Unix domain sockets is not configured (there is no support presently implemented so this change just avoids trying to compile some code that is not used)
  • configure now checks to see if the ftruncate function is present in the C library before trying to emulate it
  • the check for support of memory-mapped files was corrected; it had compiled a test application but never actually ran it
  • if the file <sys/bsdtypes.h> is present it is now automatically included in port.h
  • if the file <net/errno.h> is present it is now included by code that uses socket-related system calls
  • configure now checks for function prototypes for socket-related system calls used by the software
  • the manual page handling now supports 4.4BSD-organized pre-formatted manual pages that are not compressed

CHANGES IN THE MODEM CONFIG FILES:
  • a new prototype config file rockwell was added to deal with a problem where the rc144ac prototype file was being selected for modems based on the RC288DPI

CHANGES TO FAXSEND:
  • the code that writes files in the info directory has been rewritten
  • several error messages were corrected

CHANGES TO HFAXD:
  • various portability problems were fixed

CHANGES TO SENDPAGE:
  • sendpage is still broken

CHANGES TO MISCELLANEOUS PROGRAMS:
  • a bug was fixed that caused faxaddmodem and probemodem to not find an appropriate echo command on SunOS 4.1.4 systems
  • the faxqclean program now purges document files with a cover prefix; this fixes a problem where documents submitted by old software were not being purged
  • the tiff2fax script was fixed to correctly invoke tiffcp when converting submitted TIFF that require data reformatting

CHANGES IN THE SUPPORT LIBRARY:
  • various portability problems were fixed
  • a bug in the text to PostScript formatting class that broke the textfmt program was fixed
