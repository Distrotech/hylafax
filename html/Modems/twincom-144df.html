CLASS 1 SUPPORT
  • RTS/CTS is not honored while sending/receiving page data; use AT&K4 for software (XON/XOFF) flow control
CLASS 2 SUPPORT
  • Changing DCD handling can cause the modem to drop DSR; if done during initialization this can cause the fax server to get confused; be sure that the setting loaded from NVRAM on reset agrees with the value specified in the config file (&C1 by default);
  • Instead of the normal &D3 setup string for RC144AC-based modems, &D2 should be used (note that this configuration must be done manually as faxaddmodem is incapable of distinguishing between RC144AC-based modems)
  • The line counts in the +FPTS result code during a reception are total nonsense; the only way to deduce the number of lines in a received page of data is to parse the data on the host
  • The modem has frequently been observed to just lockup during message receive (i.e. AT+FDR); it dropped DCD to the host but did not return <DLE><ETX> and/or a +FHNG code
  • In testing the modem locked up during transmission to a Multi-Tech 1432BA modem and failed to respond to the host dropping DTR
