Supra provides documentation on the Class 1 programming interface.
Supra provides documentation on the Class 2 programming interface.
Firmware rev V1.80-02 TR14-Jxxx-001 is a significant improvement over previous firmware; it is highly recommended that you upgrade your modem if you have older firmware.
CLASS 1 SUPPORT
  • The AT+FAE command that can be used to enable auto-detect of calls from data modems is only accepted in Class 0 (data mode); I've never been able to get it to work correctly (when setup the modem is either unable to establish carrier, or the modem locks up after notifying the host)
  • RTS/CTS is not honored while sending/receiving page data; use AT&K4 for software (XON/XOFF) flow control
CLASS 2 SUPPORT
  • Changing DCD handling can cause the modem to drop DSR; if done during initialization this can cause the fax server to get confused; be sure that the setting loaded from NVRAM on reset agrees with the value specified in the config file (AT&C1 by default); one possible way of doing this is to specify &F2 for the ModemResetCmds parameter
  • The line counts in the +FPTS result code during a reception are total nonsense; the only way to deduce the number of lines in a received page of data is to parse the data on the host
  • Facsimile reception is locked at 19200 baud
  • Adaptive answer is not reliable
  • The training algorithm can loop forever; client software must timeout AT+FDR requests (fixed in rev V1.80-02 TR14-Jxxx-001)
  • RTS/CTS is not honored while receiving page data
  • The firmware often responds incorrectly to an EOM post-page message; the modem has been observed to drop carrier and/or respond with an incorrect HDLC message
