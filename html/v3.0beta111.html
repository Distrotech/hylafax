HYLAFAX CHANGE INFORMATION
    Current Version: v3.0beta111
    Previous Version: v3.0beta110
    Master FTP Site: sgi.com (192.48.153.1), directory sgi/fax

This document describes the changes made to the software between the previous and current versions (see above). If you don't find something listed here, then it was not done in this timeframe, or it was not considered important enough to be mentioned. The following information is located here:

  • Changes in the software configuration
  • Changes in the modem config files
  • Changes to faxsend
  • Changes to faxgetty
  • Changes to faxaddmodem

CHANGES IN THE SOFTWARE CONFIGURATION:
  • any specified SCRIPT_SH is now used no matter what (if it referenced an old version of bash it used to be ignored)
  • _PATH_DEV is defined if not found in <paths.h>

CHANGES IN THE MODEM CONFIG FILES:
  • the many USR config files have been consolidated

CHANGES TO FAXSEND:
  • the modem initialization commands were split to workaround a bug in the USR Sportster
  • the initialization of Class 2 and 2.0 modems was corrected to not send Class 2/2.0 commands when the modem was in Class 0
  • Class 2/2.0 commands related to setting up receive capability are no longer sent for a send operation
  • the timeout waiting for XON before sending page data was shortened
  • Dial rules can now specify ``!'' using ``\!''.

CHANGES TO FAXGETTY:
  • the modem initialization commands were split to workaround a bug in the USR Sportster
  • the Class 2 driver now shuts down faster after completing a receive

CHANGES TO FAXADDMODEM:
  • a bug in the prompting of arguments for the faxq config file was fixed
